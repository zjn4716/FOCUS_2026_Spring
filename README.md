# FOCUS_2026_Spring



本项目基于STM32F103C8T6 核心板，搭配 L9110S/HW095 电机驱动模块、TT 减速电机，完成从基础 GPIO 控制、PWM 无级调速、串口指令控制到编码器转速采集的全流程嵌入式开发。项目采用 Keil5 开发，基于 HAL 库实现，严格遵循嵌入式工程最佳实践，解决了硬件调试、驱动控制、通信交互、转速检测等核心问题，可直接应用于智能小车、小型风扇等调速场景。
开发环境与硬件清单
1. 开发环境
主开发：Keil MDK5（支持固件下载、调试） 

库支持：STM32 HAL 库 F1 版本 1.8.5

调试：ST-Link V2 下载器
辅助：USB-TTL 串口模块、杜邦线、5V 电源
工程说明
工程模版：本工程模版基于客服提供的标准工程框架修改，保留了基础工程结构，优化了 HAL 库初始化代码、中断配置、驱动函数封装，提升了代码可读性和可维护性。
接线方式：严格按照客服提供的官方接线图完成硬件连接，无自定义接线，保证硬件兼容性和调试一致性。
代码结构：模块化封装（GPIO 驱动、PWM 驱动、串口驱动、编码器驱动、指令解析），降低耦合度，便于二次开发。
任务实现详解
TASK1 物资准备、环境安装及 GPIO 电机启停
核心实现
接线参考
![示例图片](./3-1 LED闪烁.jpg"这是一个示例")
配置 2 路 GPIO (A0,A1)为推挽输出、无上下拉模式，对应 L9110S 的 IN1、IN2 引脚；
功能逻辑
高电平控制电机启动，低电平控制电机制动；
实现电机稳定启停，无卡顿、无异响、驱动模块无异常发热。
问题解决（重点）
ST-Link 接线错误，Keil 找不到设备
问题原因：ST-Link 与 STM32 接线错乱（SWDIO/SWCLK 接反、GND 未共地、3.3V 供电异常）；
解决方案：严格核对接线：ST-Link SWDIO ↔ STM32 SWDIO、ST-Link SWCLK ↔ STM32 SWCLK、共地（GND-GND）、3.3V 直连；重新安装 ST-Link 驱动，Keil 中选择正确的调试器（ST-Link Debugger），扫描设备即可识别。
硬件验证：电机启停响应迅速，硬件链路完全通畅。
TASK2 PWM 无级调速
核心实现
启用通用定时器TIM3，配置为 PWM 输出模式；
设置 PWM 频率：20kHz（电机最佳驱动频率），占空比范围 0~100%；
替代原始 GPIO 驱动，实现 PWM 平滑调速。
功能逻辑
采用非阻塞式延时（基于系统滴答定时器 SysTick），禁止使用 HAL_Delay 阻塞 CPU；
自动循环渐变：慢 → 中 → 快 → 中 → 慢，转速平滑无跳变；
解决电机直接驱动的大电流堵转问题，控制精度大幅提升。

技术亮点
非阻塞设计：不占用 CPU 主循环，可并行处理其他任务；
PWM 参数适配：匹配 TT 电机特性，启动无抖动、运行平稳。
TASK3 串口指令控制
核心实现
硬件与配置
启用 USART1 串口，配置波特率 9600，开启串口接收中断；
采用缓冲区接收机制，避免轮询方式占用 CPU 资源。
指令协议（容错解析）
表格
指令格式	功能说明
ON	启动电机（默认占空比 20%）
OFF	停止电机（占空比置 0）
SPEED 数值	设置转速（0~100 对应 0%~100%）
容错机制
参数超出 0~100 范围、指令格式错误、无效指令，串口返回ERROR；
指令解析稳定，无丢包、误触发问题。
问题解决（重点）
电机啸叫问题
问题原因：最低速度占空比过低，电机力矩不足，导致堵转啸叫；
解决方案：软件限制最低转速阈值（≥15%），提升低速力矩，啸叫问题完全解决。
TASK4 编码器转速测试
核心实现
方案选择：采用定时器编码器模式（TIM2）
优势：四倍频解码、采样精度高、CPU 资源占用极低，规避外部中断高频占用 CPU 的风险；
测速原理
编码器每转输出固定脉冲数，通过定时器读取脉冲计数值；
单位时间换算电机实际转速（RPM），实现实时采集。

串口实时打印当前电机转速（RPM）；
调速与转速采集同步运行，数据稳定无漂移。
